### A11Y-ALT-TEXT â€” Images must have meaningful alternative text

Every informative image must have an `alt` attribute conveying the same information. Decorative images must use `alt=""`. An absent `alt` attribute is always a violation.

Violations to detect:

- `<img>` elements with no `alt` attribute
- `<img alt="">` where the image conveys meaningful content (charts, people, action icons)
- `alt` text that is a filename, URL, or generic phrase (`"image"`, `"photo"`, `"icon"`)
- Icon-only buttons or links with no `aria-label` and no visible adjacent text
- `<input type="image">` without an `alt` describing the button action
- CSS `background-image` used for content images rather than decoration (no alt possible)

### A11Y-SEMANTIC-HTML â€” Use semantic HTML elements; do not build interactive controls from divs

Use the most semantically precise HTML element. Do not use `<div>`/`<span>` with click handlers in place of `<button>`, `<a>`, or other native interactive elements.

Violations to detect:

- `<div>` or `<span>` with `onClick` but no `role` and no `tabIndex`
- Page layout with no landmark elements (`<main>`, `<header>`, `<footer>`, `<nav>`)
- `<div>` used as a button or navigation item via CSS class naming alone
- Headings using `<div>` or `<span>` styled with heading font sizes rather than `<h1>`â€“`<h6>`
- Form controls built from `<div>` without correct ARIA roles and associated labels
- Tables used for layout rather than tabular data

### A11Y-KEYBOARD-NAVIGATION â€” Interactive elements must be fully keyboard-accessible

All interactive functionality must be reachable and operable using only a keyboard.

Violations to detect:

- `onClick` on a non-native interactive element (div, span) without `onKeyDown`/`onKeyPress` and `tabIndex`
- `tabIndex="-1"` on elements that are the only way to trigger an action
- `onMouseEnter`/`onMouseLeave` used to reveal content with no `onFocus`/`onBlur` equivalent
- Custom widgets (dropdowns, tabs, comboboxes) with no arrow-key navigation
- Modal dialogs that do not trap focus within the dialog while open
- `outline: none` or `outline: 0` with no visible replacement focus indicator

### A11Y-COLOR-CONTRAST â€” Text must meet minimum contrast ratio against its background

Normal text needs â‰Ą 4.5:1 contrast; large text and UI component boundaries need â‰Ą 3:1. Audit-scope is limited to statically detectable hardcoded colour values.

Violations to detect:

- Light grey text on white (`#999`, `#aaa`, `#bbb`) which falls below 4.5:1
- Text with `opacity` below 0.5 reducing effective contrast without background consideration
- Placeholder text at significantly reduced opacity with no fallback styling
- `color` and `background-color` set to visually similar values in the same CSS rule
- Focus indicators using low-contrast colours against adjacent content
