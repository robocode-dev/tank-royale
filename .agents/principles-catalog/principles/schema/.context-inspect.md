# .principles inspect context — schema
# Machine-executable pre-scan patterns per principle

### SCHEMA-FIELD-OPTIONALITY

- `grep -rnE '"required"\s*:\s*\[' --include="*.json" --include="*.yaml" --include="*.yml" $TARGET` | MEDIUM | JSON Schema / OpenAPI required array — verify no new fields added to published schemas
- `grep -rnE '^\s+\w+\s+\w+\s*=\s*\d+\s*;' --include="*.proto" $TARGET` | INFO | Protobuf field — verify optionality and default semantics

### SCHEMA-NO-POLYMORPHIC-BLOBS

- `grep -rnE 'google\.protobuf\.(Any|Struct|Value)' --include="*.proto" $TARGET` | HIGH | Protobuf Any/Struct/Value bypasses typed schema
- `grep -rnE 'additionalProperties:\s*true' --include="*.yaml" --include="*.yml" --include="*.json" $TARGET` | MEDIUM | OpenAPI additionalProperties: true allows unconstrained fields
- `grep -rnE 'type:\s*object\s*$' --include="*.yaml" --include="*.yml" $TARGET` | MEDIUM | OpenAPI object type without properties — unconstrained schema

### SCHEMA-ENUM-EVOLUTION

- `grep -rnE '^\s*enum\s+\w+\s*\{' --include="*.proto" -A 2 $TARGET` | MEDIUM | Protobuf enum — verify first value is UNSPECIFIED/UNKNOWN (field number 0)
- `grep -rnE 'enum:\s*\[' --include="*.yaml" --include="*.yml" --include="*.json" $TARGET` | MEDIUM | OpenAPI enum — verify UNKNOWN variant or non-exhaustive documentation
