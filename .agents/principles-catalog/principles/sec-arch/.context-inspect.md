# .principles inspect context — sec-arch
# Machine-executable pre-scan patterns per principle

### SEC-ARCH-SUPPLY-CHAIN-SECURITY

- `grep -rnE 'uses:\s+[^@]+@(main|master|latest|v[0-9]+)$' --include="*.yml" --include="*.yaml" $TARGET` | HIGH | GitHub Actions pinned to mutable ref
- `grep -rnE '^FROM [a-zA-Z0-9_./:-]+:[a-zA-Z0-9_.-]+$' --include="Dockerfile" --include="*.dockerfile" $TARGET` | MEDIUM | Docker image pinned by tag not digest

### SEC-ARCH-ECONOMY-OF-MECHANISM

- `grep -rnE '(AES|DES|RSA|SHA[0-9]|HMAC|encrypt|decrypt|cipher)\s*[\(\.]' --include="*.java" --include="*.py" --include="*.ts" --include="*.go" --include="*.cs" $TARGET` | MEDIUM | Potential hand-rolled cryptography — verify library usage

### SEC-ARCH-LEAST-COMMON-MECHANISM

- `grep -rnE '(static|global|shared).*(token|session|credential|auth|secret)' --include="*.java" --include="*.py" --include="*.ts" --include="*.go" --include="*.cs" $TARGET` | MEDIUM | Potential shared security state in global/static variable

### SEC-ARCH-OPEN-DESIGN

- `grep -rnE '(secret|password|token|api_key|apikey)\s*=\s*["'"'"'][^"'"'"']{8,}' --include="*.java" --include="*.py" --include="*.ts" --include="*.go" --include="*.cs" --include="*.rb" --include="*.env" $TARGET` | HIGH | Hardcoded secret used as security mechanism
- `grep -rnE '(security|safe|secure|protected).*obscur|obscur.*(security|safe|secure|protected)' --include="*.md" --include="*.txt" --include="*.java" --include="*.py" --include="*.ts" $TARGET` | MEDIUM | Reference to security through obscurity
