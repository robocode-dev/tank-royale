# Docs Stack — Layer 1: Universal Principles

These principles are **always active** for any documentation file, regardless of type or content. They apply to README files, architecture docs, guides, how-tos, and any other prose artifact.

Note: `SIMPLE-DESIGN-REVEALS-INTENTION`, `CODE-CS-DRY`, `CODE-CS-KISS`, `CODE-CS-YAGNI`, `CODE-DX-NAMING`, and `ARCH-DECISION-RECORDS` are activated by the universal section of `artifact-types.yaml` and apply to all stacks including docs.

| ID | Title | Summary |
|----|-------|---------|
| DOC-PURPOSE | Every document has one clear purpose | A document should serve exactly one reader goal. If a document tries to be a tutorial and a reference simultaneously, it fails at both. |
| DOC-MINIMAL | Write only what is necessary | Apply YAGNI to prose: do not document speculative features, rarely-used edge cases, or information the reader can derive from context. |
| CODE-DX-CODE-FOR-READERS | Write for the reader, not the writer | Prioritize clarity over completeness. Every sentence should serve the reader's understanding, not the author's need to record information. |
| CODE-DX-REDUCE-COGNITIVE-LOAD | Reduce cognitive load | Minimize the effort required to understand the content. Use familiar structure, short paragraphs, clear headings, and progressive disclosure. |
