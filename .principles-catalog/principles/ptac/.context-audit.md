# Plain-Text-as-Code principles by flemming-n-larsen.

### PTAC-PLAIN-TEXT-FIRST

**Principle:** Prefer plain text formats over binary; keep every artifact diffable and human-readable.

**Violations to detect:**
- Committing `.docx`, `.xlsx`, `.pdf`, or other binary office formats as the source of truth
- Binary serialization formats (`.ser`, `.pkl`) used for configuration or data files
- Images or diagrams in binary formats where SVG or Mermaid would suffice
- Build artifacts, compiled outputs, or generated binaries committed alongside source

### PTAC-COMPOSABLE-FILES

**Principle:** Keep files small, focused, and composable rather than large and monolithic.

**Violations to detect:**
- Files exceeding 300 lines without clear structural justification
- Configuration or documentation files mixing multiple unrelated concerns
- Infrastructure modules provisioning unrelated resource types in a single file
- Workflow files with more than 5 logical stages that could be split
- README files acting as reference docs, tutorial, and changelog simultaneously

---

### PTAC-NO-GENERATED-BLOBS

**Principle:** Commit source and generation instructions, not generated or binary output blobs.

**Violations to detect:**
- Compiled binaries (`.jar`, `.exe`, `.wasm`) committed to the repo
- Generated API clients or SDKs checked in without a `# generated` marker
- Generated output that is out of sync with its source (e.g., stale compiled CSS)
- AI-generated content committed wholesale without review

---

### PTAC-NAVIGABLE-DIRS

**Principle:** Every plain-text directory must contain `INDEX.md` (structured content list for AI/automation) and `README.md` (human description, rendered on GitHub).

**Violations to detect:**
- Plain-text directory with no `README.md` (no human entry point, invisible on GitHub)
- Plain-text directory with no `INDEX.md` (AI must guess structure from filenames alone)
- A single file trying to serve as both index and readme (two concerns mixed)
- `README.md` and `INDEX.md` that duplicate each other — same tables or descriptions in both (DRY violation)
- Subdirectory added without updating the parent's `INDEX.md`
- `INDEX.md` that describes purpose instead of listing files (wrong role)
- `INDEX.md` that describes purpose instead of listing files (wrong role)

---

### PTAC-LEAN-CONTENT

**Principle:** Keep plain-text content DRY and front-loaded; every line must earn its place.

**Violations to detect:**
- Key decision, constraint, or summary buried after lengthy preamble
- The same information stated in multiple files without a canonical source
- Background or context section longer than the actionable content it introduces
- Filler sentences that don't change understanding ("This section describes...", "As mentioned above...")
- Files over 200 lines that could be split or trimmed without losing meaning
